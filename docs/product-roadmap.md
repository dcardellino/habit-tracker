## Build Philosophy

1. **Each phase ships something usable.** No phase leaves the app broken. After any phase, you can demo what you've built.
2. **Infrastructure before features.** Phase 0 sets up everything — Convex, Auth, Tailwind, shadcn/ui — so feature work in later phases is fast and clean.
3. **Magic moment first.** By the end of Phase 1, a user can open the dashboard and see all habits with streaks in one view. Everything else builds on top.
4. **Test as you go.** Each task ends with a "Verify:" note describing how to confirm it works before moving to the next.
5. **Mobile-first responsive.** Every screen is tested on mobile web (iOS Safari, Android Chrome) from Phase 0. The primary use case is a morning check-in on a phone.
6. **Review before proceeding.** After each phase, push a PR and run a review agent (e.g. [CodeRabbit](https://coderabbit.ai)) before starting the next phase. Catching issues phase-by-phase is much cheaper than catching them in Phase 3.
7. **Design tokens are the foundation.** Run `/plaid design` with streak.kevinchromik.de as reference and generate `docs/design.md` before any UI work. Do not improvise colors, typography, or spacing — use the token system.

## Agent Session Guide

### How to Use This Roadmap with Your Coding Agent

1. **Start a session:** Give your coding agent the phase prompt at the beginning of each phase.
2. **Read selectively:** Each phase lists its Reference sections — read only those, not the full documents.
3. **Let it work:** The agent reads the roadmap, finds the first unchecked task, implements it, marks it complete.
4. **One session per phase (ideally):** Try to finish a full phase per session. If you stop mid-phase, resume from the first unchecked task.
5. **Push a PR for review:** After each phase, push to a `phase-N/slug` branch and open a PR. Run [CodeRabbit](https://coderabbit.ai) (free for public repos) before starting the next phase.
6. **Update the status line:** After completing tasks, update the header: `**Status:** X/47 tasks complete`.

### Session Tips

- **Don't skip tasks** — they're ordered intentionally. Skipping creates dependency issues.
- **Verify after each task** — each task has a "Verify:" note. Don't move to the next task until verification passes.
- **Run `/plaid design` before Phase 0 UI work** — the design system must exist before any styling decisions.
- **Mobile-first** — test on mobile after every screen implementation. Don't leave it for Phase 3.
- **Timezone is tricky** — validate streak logic edge cases explicitly. Don't assume UTC.
